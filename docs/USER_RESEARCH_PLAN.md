# CAPSULA User Research Plan

## Research Goal

Validate whether CAPSULA helps real users move from idea to working app capsule faster than their current workflow.

## Target Participants

- 5 technical founders
- 3 agency builders
- 3 AI-native developers
- 2 educators
- 2 research/lab users

## Study Tasks

1. Open the web studio.
2. Explain what CAPSULA does after 60 seconds.
3. Create a Python capsule.
4. Generate a manifest.
5. Create an Expo Go capsule.
6. Find the deploy plan.
7. Open an issue using the issue-bot template.
8. Describe whether they would use this for a real project.

## Questions

- What part felt most real?
- What part felt confusing?
- What did you expect the Run button to do?
- Did the capsule concept make sense?
- Would Expo Go preview help you demo to clients or investors?
- Which runtime lane matters most to you?
- What would make this worth paying for?

## Metrics

- time to first understanding
- time to first capsule
- time to first manifest
- Expo generation success
- perceived trust score
- willingness to pay
- top requested runtime

## Success Criteria

CAPSULA is alpha-user ready when:

- 70% of testers understand the capsule concept in under 2 minutes
- 60% can create a capsule using docs
- 50% say they would use it for a real project
- at least 3 users ask for hosted preview or team features

## Research Artifacts

- session notes
- screen recordings
- issue links
- generated capsule manifests
- feedback summary
- roadmap updates
