# CAPSULA Commercial Readiness Checklist

## Product Readiness

- [x] Commercial web interface exists.
- [x] Runtime lanes are visible.
- [x] Python runtime package exists.
- [x] API server exists.
- [x] Preview server exists.
- [x] MCP-style tool server exists.
- [x] Expo Go generator exists.
- [x] WASM planner exists.
- [x] Issue templates exist.
- [x] Issue-bot operating issues exist.
- [x] Whitepaper exists.
- [x] Technical paper exists.
- [x] Investor brief exists.
- [x] Product requirements exist.

## Still Needed Before Paid Users

- [ ] Connect web app buttons to API calls.
- [ ] Add Monaco editor.
- [ ] Add file tree.
- [ ] Add persistent session database.
- [ ] Add WebSocket logs.
- [ ] Add hosted preview.
- [ ] Add branch/PR deploy workflow.
- [ ] Add authentication.
- [ ] Add billing/waitlist page.
- [ ] Add analytics.

## Demo Readiness

A funding or customer demo should show:

1. Web app running.
2. Runtime lane selection.
3. Python session command.
4. Manifest generation.
5. Expo Go mobile generation path.
6. Issue bot roadmap.
7. Research papers.
8. Investor brief.
9. GitHub commits.

## Release Gate

Before calling a version commercially ready:

- CI passes.
- Web app builds.
- Python tests pass.
- Issue bot docs are current.
- README has correct commands.
- No broken first-run command paths.
- Security limitations are documented.
- Toolchain-required lanes are clearly marked.

## Funding Readiness

CAPSULA is fundable when the demo shows:

- clear problem
- clear product wedge
- working interface
- working local backend
- mobile preview story
- AI/MCP story
- issue-bot operating model
- credible roadmap
- early user path
- commercial packaging
