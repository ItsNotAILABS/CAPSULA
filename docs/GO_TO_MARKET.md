# CAPSULA Go-To-Market Plan

## Positioning

CAPSULA Studio is the commercial app for turning AI-assisted coding sessions into deployable capsules: apps, agents, workers, mobile previews, and release packages.

## Beachhead Users

1. Technical founders building MVPs.
2. Agencies building client prototypes.
3. AI-native developers building internal tools.
4. Educators teaching full-stack app creation.
5. Research labs packaging experiments.

## Core Message

Do not just generate code. Build a capsule you can run, preview, inspect, and deploy.

## Launch Offer

CAPSULA Alpha:

- local studio
- Python runtime
- web preview
- Expo Go generator
- MCP AI bridge
- GitHub deploy handoff
- issue-bot workflow

## Acquisition Channels

- GitHub repo and demos
- X/Twitter build-in-public posts
- short demo videos
- founder outreach
- agency partnerships
- AI developer communities
- Expo and React Native communities
- WebAssembly developer communities

## First Demo Script

1. Open CAPSULA web studio.
2. Select Python Agent capsule.
3. Create session.
4. Run capsule.
5. Generate manifest.
6. Switch to Expo Go capsule.
7. Generate mobile app.
8. Show QR preview path.
9. Open issue-bot release checklist.
10. Show GitHub deploy plan.

## Investor Demo Goal

Make CAPSULA feel real within the first 60 seconds:

- professional interface
- clear runtime lanes
- local server commands
- mobile preview story
- issue-bot workflow
- research papers and docs
- GitHub commits as execution proof

## Risks

- too broad runtime scope
- users may expect cloud hosting immediately
- local toolchains can be confusing
- issue bots may be perceived as roadmap rather than automation

## Mitigation

- lead with Python, React, Expo, GitHub
- be honest on toolchain-required lanes
- add hosted preview next
- show real commands and repo evidence
