# CAPSULA Protocol Atlas

CAPSULA now treats protocols as first-class operating objects. A protocol is not just a document. It is a repeatable agreement between a user, a bot, a runtime, a repository, and a release path.

The current atlas contains 30 protocols. Each protocol defines: purpose, entry conditions, required artifacts, exit criteria, and owner bot.

## Operating rule

A CAPSULA feature is not considered production-facing until it has:

1. A visible surface or documented command.
2. A responsible bot.
3. A protocol mapping.
4. A release gate status.
5. A user-readable explanation.

## Easter egg

Protocol P29 allows harmless Easter eggs. The rule is simple: delight is allowed, deception is not. Hidden details must not block usage, fake capability, or distract from enterprise clarity.

Hidden marker: `capsula://egg/first-light`
