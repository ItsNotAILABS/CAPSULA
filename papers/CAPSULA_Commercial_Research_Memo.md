# CAPSULA Commercial Research Memo

## Market Hypothesis

CAPSULA targets the gap between AI code generation and production-grade product delivery. The likely early market is made of builders who already use AI coding tools but still struggle with packaging, previewing, explaining, and shipping usable apps.

## Initial Customer Segments

### 1. Solo AI builders

They need a path from prompt to usable prototype without spending days wiring local servers, mobile previews, and deployment instructions.

### 2. Consultants and agencies

They need repeatable project packaging for client-facing prototypes, internal tools, and mobile demos.

### 3. Technical founders

They need investor-facing evidence: screenshots, architecture, roadmap, release plan, security posture, and user research.

### 4. Product teams

They need governed AI-assisted building with release gates, issue triage, team workspaces, and audit trails.

## Buyer Pain

The pain is not only code generation. The pain is the distance between generated code and a trustworthy app that can be shown to users, buyers, or investors. Users need:

- quick preview,
- clean workspace,
- runtime clarity,
- deploy checklist,
- mobile demo path,
- release evidence,
- docs and user guide,
- security explanation.

## Wedge

The wedge is the capsule: one repeatable object that packages code, runtime, preview, manifest, workers, and deploy plan.

## Activation Metric

The first product activation metric should be:

> A user creates a capsule, opens preview, generates a manifest, and reviews a deploy plan in one session.

This is stronger than signups or file generation because it proves the product delivered a runnable workflow.

## Funding Narrative

CAPSULA can be positioned as an AI-native software production studio. It is not only an IDE. It is a productization layer for AI-generated software.

Investors should understand four defensible directions:

1. Workflow memory: each capsule retains build and deploy evidence.
2. Runtime breadth: multiple language lanes share one product surface.
3. Trust layer: release gates and issue bots reduce fake claims.
4. Distribution: Expo Go, GitHub, and web previews make outputs immediately shareable.

## Near-Term Research Questions

- Which user segment reaches activation fastest?
- Which runtime lane creates the most perceived value?
- Does Expo Go preview increase user willingness to share prototypes?
- Do release gates increase buyer confidence?
- How much of the workflow can remain local-first before hosted infrastructure is required?

## Conclusion

CAPSULA's commercial opportunity is to make AI-generated software feel operational, inspectable, and shareable. The business is strongest if the platform proves faster time-to-first-running-app while increasing user trust in the output.
